## Workflow (local-verify-primary)

- Single-agent serial by default; independent clones only for genuinely
  parallel sub-agents.
- Branch per chip off `main`; merge gate = throttled full `lake build`
  green locally OR Lean Action CI green on the branch (CI is the gate while
  the local cache is cold). NEVER push unverified code to `main`.
- Single-file `lake env lean` does NOT catch cross-file duplicate names or
  stale-import signature drift ‚Äî any new top-level declaration needs the
  full-graph gate before merge.
- `set_option diagnostics true` stays on while debugging; read the actual
  error at the bottom of stdout; debug in place, never delete-and-restart.
- No `sorry`, no `axiom` in anything intended for submission. The vacuity
  linter runs as a merge gate (see `CHIP_GATES.md`) ‚Äî "zero sorry" alone is
  NOT the bar; sqg-lean-proofs had zero sorries while circular.

  writing infrastructure (after bootstrap; until then use the pinned file
  inventory in `docs/ROUTE_RESEARCH_2026_06_13.md`).

  `/Volumes/4TB SD/ClaudeCode/references/proof_obligation_ledger_schema.md`:
